## 2. アーキテクチャ

### 2.1 `gui/strings.py` — GUI 文言の単一の真実

`src/valisync/gui/strings.py` を新設。**pure Python・Qt 非依存**（`theme/tokens.py` と同じ隔離方針 — headless で全テスト可能）。

- **形式**: モジュールレベル `Final[str]` 定数。エリア別プレフィックスで命名（`MENU_FILE = "ファイル(&F)"`・`DOCK_FILE_BROWSER`・`DIALOG_EXPORT_TITLE`・`STATUS_LOADED = "{source} を読み込みました"` 等）。dict キー参照ではなく属性参照 — typo が mypy/ImportError で loud-fail し、実行時 KeyError の芽がない。
- **テンプレート**: 動的文言は名前付きプレースホルダの `str.format` テンプレート定数（`"{source} を読み込みました"`）。呼び出し側で `.format(source=...)`。書式付き数値（`{offset:+.3f}`）はテンプレートに書式を内蔵し、散在フォーマッタを引き寄せる（R-06 のオフセット単一化）。
- **重複定義の解消**: 現状の三重/二重定義（File Browser×3面・Diagnostics×3面＋参照文2・テーマラベル×2・scope radio・区切りコンボ 2 ファイル）は全て同一定数の参照に置換。ドック名は windowTitle／折りたたみタイトルバー／レールタブ／参照文（「診断パネルを参照」等）が**同じ定数**から合成される。
- **QTranslator を狭めない**: キー付き定数テーブルは将来 `QT_TRANSLATE_NOOP`/gettext へ機械変換可能な形。今回は翻訳フレームワークを導入しない（YAGNI — 対象言語は日本語1つ）。

### 2.2 Qt 標準ボタンの日本語化 — qtbase QTranslator（判断点 #10 確定）

`theme/apply.py` の起動適用（`apply_startup_theme` 系列）に **qtbase 日本語 QTranslator のロード**を追加:

- `QLibraryInfo.path(QLibraryInfo.LibraryPath.TranslationsPath)` から `qtbase_ja` をロードし `QApplication.installTranslator`。PySide6 wheel は `translations/qtbase_ja.qm` を同梱している（実装タスクで実在をテスト検証）。
- これで QDialogButtonBox（OK/Cancel/Yes/No→OK/キャンセル/はい/いいえ）・QFileDialog chrome・テキスト編集の標準コンテキストメニュー等が一括日本語化される（根本解決 — 個別 setText の網羅漏れが構造的に起きない）。
- **翻訳者オブジェクトの生存保持**: `QTranslator` は install 後も Python 参照を保持しないと GC で外れる（[[gui_qrunnable_signals_lifetime_retention]] と同型の罠）— module-level またはアプリ属性で保持。
- 標準ボタンでも文言を明示制御したい箇所（例: 削除確認の「閉じる/キャンセル」）は従来どおり `button().setText(strings.X)` で上書き — QTranslator と両立。

### 2.3 core 診断・例外文言 — core 内リテラル日本語（UX-08）

core は gui へ依存できない（レイヤー逆転禁止）ため、`strings.py` へは載せず **core 内のリテラルを日本語化**する:

- 対象はインベントリ core クラスタ 32 件（mdf_loader / csv_loader / session / format_def / csv_exporter / format_def_manager のユーザー露出文言）。
- 診断プレフィックスは `信号 '{name}':` へ 6 箇所統一（G-03）。`signal_name` フィールドは別途構造化保持済みのため表示文言のみの変更で安全。
- 2 ローダーで同一原文の文言（`File not found or not accessible: {path}`）は同一訳を強制 — **同一性テスト**（両モジュールの該当定数/文言が一致することを assert）で writer のコピー先揺れを恒久防止。
- 検証系 ValueError（format_def.py・csv_exporter.py — ダイアログのエラーラベルへ str(exc) 生表示）も core 側で日本語化（判断点 #11 確定）。**文言依存テストの同時更新必須**（mdf_loader.py:440 の互換注記が既知例 — 実装プランで文言 grep により全数列挙）。

### 2.4 ニーモニクス体系（UX-41）

- 全メニュー項目・主要ボタンに「ラベル(&X)」形式（半角括弧・ラベル末尾）でアクセスキーを付与。トップレベル: ファイル(&F)・表示(&V)・解析(&A)・ヘルプ(&H)。
- **重複検出テスト常設**: `strings.py` にメニュー単位のグルーピング（`MENU_FILE_ITEMS: tuple[str, ...]` 等のタプル定数）を持たせ、Layer A テストが各グループ内の `&` 直後文字の重複を検出する。文言変更でキーが衝突したら CI が落ちる。
- サブメニュー項目はニーモニクス任意（衝突回避優先）。三点リーダとの位置関係は「開く(&O)…」（R-03）。

### 2.5 変更しないもの（互換性の錨）

- **objectName・QSettings キー**: dock/widget の objectName は `saveState()`/`dockCollapsed` 永続化の識別子 — 一切変更しない（windowTitle 等の表示文言のみ変更）。
- **VM キー・データ値**: `visible_stat_cols` の VM キー（mean/min/max/std/count）・csv_format_dialog の currentText データ値兼用（sec/msec）は据え置き（判断点 #4/R-11）。表示と内部キーの分離導入は据え置き理由ごと対訳表に「意図的英語」として記録。
- **数値フォーマットの役割分担**: 時刻ヘッダ `.4g`・readout 値列 precision（4/6/8）・オフセット `+.3f`・カーソル即値 `A {t:.3f} s` は現行仕様のまま（計測 IA 増分で確定済み）。

## 4. 表記規約（docs/design.md へ「表記規約」節として転記 — UX-55 の恒久解）

| # | 規約 |
|---|---|
| R-01 | **一次言語は日本語**。原文維持: 固有名詞（ValiSync）・頭字語/規格名（CSV/MDF/XCP/OK）・単位（s/Hz）・記号（Δ/σ/±/A/B）・キー名（Ctrl/Shift/←→）・識別子/ファイル名/dtype/例外原文（{exc}）。 |
| R-02 | **括弧**: 括る内容が ASCII のみ（ショートカット・記号・数値）なら半角括弧＋直前半角スペース（「カンマ (,)」「新規タブ (Ctrl+T)」）。日本語または非 ASCII 記号（Δ 等）を含む補足は全角括弧（「カーソル B（Δ）」「（履歴なし）」）。全角括弧の RUF001 は noqa 付与を既定とし、lint 回避のために表記を歪めない。 |
| R-03 | **三点リーダ**: 後続入力・ダイアログを伴うアクションのみ末尾「…」（U+2026 1文字）。即時実行には付けない。ニーモニクスは三点リーダの前（「開く(&O)…」）。 |
| R-04 | **ショートカット表記**: QAction は setShortcut 登録で Qt 自動併記に任せ、ラベルへの手書き併記は禁止（welcome_view.py の「(Ctrl+O)」stale 罠はボタン文言を action から動的合成して根治）。ボタン/ツールチップへ併記する場合は「ラベル (Ctrl+O)」— 半角括弧・区切りスペース1つ・キーは + 前後スペースなし。 |
| R-05 | **ダッシュ**: 区切りは em ダッシュ「 — 」（前後半角スペース。「{name} — ValiSync」「信号プレビュー — {key}」）。数値範囲は en ダッシュ「–」スペースなし（0–255・min–max）。 |
| R-06 | **数値・単位**: 数値と単位の間に半角スペース（「+0.500 s」「1024 列」「5 件」）。オフセット表示は単一テンプレート `{:+.3f} s` を適用/入力/情報行の3画面で共用（UXG-04 の「+X.XXXs」是正）。 |
| R-07 | **助数詞**（UX-11）: 診断・ヒット件数=「件」／信号総数=「N 信号」・フィルタ結果=「{n} 信号中 {m} 件を表示」の単一パターン／展開系列・チャンネル=「本」／列=「列」／サンプル=「点」／非単調=「箇所」。「ch」は技術メタ情報表示に限定。数詞は半角数字＋前後半角スペース（「1信号」直結は禁止）。 |
| R-08 | **動詞形**: メニュー/ボタン=辞書形「〜を開く」（文脈自明の単独動詞「削除」「非表示」許容）／完了=「〜しました」／進行=「〜中…」／失敗=操作の不成立は「〜できませんでした」・内部要因＋詳細併記は「〜に失敗しました: {exc}」／依頼=「〜してください」／確認=「〜しますか？」。ボタン対は動詞で対称に（G-19）。 |
| R-09 | **入口動詞と処理動詞の分離**: 「開く」=ユーザー起点の入口ラベル／「読み込む」=処理の記述（進行・完了・失敗）。機能名は「エクスポート」で統一。 |
| R-10 | **記号・句読点**: 疑問符は全角「？」。コロンは半角「: 」＋後スペース。ダイアログ本文は「。」終止・ラベル/メニュー/ツールチップは句点なし。ステータスの列挙連結は「 ・ 」。二重スペースは即修正。診断 # 表示は 1 始まり。 |
| R-11 | **データ値と表示文字列の分離**: 表示文字列を内部キー/データ値と兼用しない。既存の兼用（csv_format_dialog の sec/msec・readout の _STAT_COLS）は「意図的英語」として対訳表に記録し据え置き（翻訳する日が来たら表示写像を先に導入）。 |
| R-12 | **カタカナ長音**: 省略式（ヘッダ・エクスプローラ・ブラウザ）★#3 — 既存文言の大勢・工学系慣行・最小 diff。決定後は個別語を対訳表に固定し揺れの再生産を防ぐ。 |

## 6. テスト戦略（/gui-test-plan 分析）

**変更種別**: 文言（表示文字列）全域置換＋起動時 QTranslator 適用。挙動・レイアウト構造・入力経路は不変（E-2/E-3 の微修正を除く）。

- **Layer A（必須・CI）**:
  - `strings.py` のニーモニクス重複検出（メニューグループ単位・§2.4）。
  - テンプレート整合（`STATUS_LOADED` 等の全テンプレート定数が期待プレースホルダ集合で format 可能）。
  - core 2 ローダーの同一文言の同一性テスト（G-34）。
  - 診断 # 1 始まり（E-2）の表示値テスト。
  - **既存テストの文言追随**: 現行文言を assert する全テストを文言 grep で列挙し同時更新（mdf_loader.py:440 互換注記が既知例。実装プランに全数リストを含める）。
- **Layer B**: 不要（イベント経路の変更なし）。E-3 の CTA 動的合成のみ、QAction shortcut 変更がボタン文言へ反映されることを widget レベルで検証。
- **Layer C（realgui・merge 前 ①ゲート）**:
  - realgui はメニュー/ボタンを**表示文言で検索**する箇所が多数 — 文言変更は掴み点崩壊としてハングや誤クリックで現れる（[[gui_behavior_change_stale_parallel_realgui_test]] の教訓: 旧文言を tests/ 全体で grep し全数追随）。
  - ①ゲート: 主要ジャーニー（開く→メニュー操作→右クリック→エクスポート→診断）を実 OS 入力で通し、**日本語化後のメニュー・ダイアログ・ステータスバーのスクショ目視**（QTranslator の効き＝QDialogButtonBox が日本語になっていることを実機で確認 — headless では translator ロード成否の false-green があり得る）。
- **凍結検証（撮影）**: 文言全域置換のため**全カタログ状態のピクセル差分が想定どおり出る**。手順: (1) 変更前後の撮影比較で**差分がテキスト領域に限られる**こと（プロット波形・レイアウト構造は不変）を確認 → (2) 新ベースラインへ一括昇格 → (3) 再撮影で compare exit 0（決定性）→ (4) DesignSync で Ground Truth 再同期。

## 7. リスクと対策

| リスク | 対策 |
|---|---|
| realgui の文言掴み点崩壊（ハング/誤クリックとして発現） | 実装プランで旧文言→新文言の**全数対応表**を作り tests/ を機械的 grep 置換。①ゲートで実機通し |
| QTranslator の波及が想定超（QFileDialog 等の見た目変化） | 対象は qtbase のみ・撮影比較で全状態を目視。想定外があれば translator を該当 context で上書き |
| `qtbase_ja.qm` が PySide6 wheel に無い環境 | ロード成否をチェックし、失敗時は警告ログのみで起動継続（文言 OS の自前文言は translator 非依存）。CI にロード成功テストを常設 |
| QTranslator の GC 消失 | 参照保持（§2.2）＋「install 後に QDialogButtonBox 文言が日本語」の Layer A テスト |
| 文言変更とテスト更新の非同期でレビュー困難 | subagent-driven のタスク分割を「エリア単位（文言＋テスト＋realgui 追随を同一タスク）」で切る |
| ドック名変更で QSettings 復元が壊れる | objectName 不変（§2.5）— windowTitle のみ変更。復元テストは既存が担保 |
| 全角括弧・記号の RUF001/RUF002 大量発生 | R-02 の noqa 既定を strings.py 冒頭で `# ruff: noqa: RUF001, RUF002` ファイル指定にし個別ノイズを排除 |

## 8. 実装増分の分割方針（writing-plans への入力)

1. **基盤**: `strings.py` 新設＋ニーモニクス/テンプレートのガードテスト＋QTranslator 適用（apply.py）。
