## 1. 背景・目的

### 現状（実コードで確認）
- **R14（時間オフセット）/ R16（Delta_Cursor）/ R17（範囲統計）は GUI に未実装**（R15 Global_Cursor は増分A で実装済み）。

## 3. 用語

| 用語 | 意味 |
|---|---|
| Global_Cursor（メインカーソル A） | パネルに1本の垂直線。時刻 A を指し各信号の補間値を表示。右クリックの「メインカーソル」トグルで表示、線の D&D で移動。 |
| Delta_Cursor（サブカーソル Δ・B） | 右クリックの「サブカーソル（Δ）」トグルで追加される2本目の垂直線（時刻 B）。A→B の差分を計測。 |
| アクティブ波形 | オフセット操作の対象として掴まれた1曲線（`_items` の1キー＝信号名）。PR #19 のアクティブ軸に相当する非永続 UI 状態。 |
| フロート表 | プロット上にオーバーレイする読み取り面。カーソル表示に連動して出没。凡例（色↔信号名）を兼ねる。 |
| 統一タイムライン | `file_offsets`/`signal_offsets` を適用して時刻同期した共通時間軸。 |

## 4. 操作モデル（プロット内側ジェスチャの交通整理）

既存責務（左 Y ストリップ＝アクティブ軸操作・下 X ストリップ＝常時ズーム/パン）は**変更しない**。カーソルの表示/非表示は右クリックメニューのトグル、移動は線の D&D に一本化する（増分B 改訂・2026-06-29 合意）:

| 操作 | 動作 | 要件 |
|---|---|---|
| **右クリック →「メインカーソル」トグル** | Global_Cursor (A) の表示 ON/OFF。ON で既定位置（可視幅 50%）に出現、OFF で A（と B）を消去 | R15.1（改訂） |
| **右クリック →「サブカーソル（Δ）」トグル** | Delta_Cursor (B) の表示 ON/OFF。ON で既定位置（可視幅 75%）に出現し Δ/統計を有効化、OFF で B・統計列を消去。**メイン ON の間だけ選択可**（OFF 時グレーアウト） | R16.1 |
| **カーソル線上でドラッグ（D&D）** | その線（A または B）を移動。**移動はこの D&D のみ**（空クリック設置は廃止） | R15.4 / R16.4 |
| **曲線上で押下＋水平ドラッグ** | その曲線をアクティブ化し時間オフセット（プレビュー）。リリースで適用ダイアログ | R14.1 |
| **右クリック →「補間方式」** | 補間方式サブメニュー（R15 既存） | R15.4 |

- **判定優先順位（ドラッグ開始時）**: カーソル線（許容ピクセル内）＞ 曲線（最近傍・許容ピクセル内）。空白上のクリック/ドラッグはカーソル操作に使わない（設置は廃止）。
- **表示の出入りはメニュートグルに一本化（R15 改訂）**: 空クリック設置を廃し、カーソルの ON/OFF は右クリックの「メインカーソル」「サブカーソル（Δ）」チェックで行う。移動は線の D&D のみ。データ改変系（オフセット）は曲線上ドラッグに限定して誤爆を防ぐ。
- 曲線ヒットテストの許容値・カーソル線ヒットの許容ピクセル・アクティブ波形のハイライト表現は**実装時に確定**（§12）。

### カーソルモデル（加算式）
- **Global_Cursor (A)** が起点。「メインカーソル」トグル ON で可視幅 **50%（中央）** に出現。表示中は常に A における各信号の補間値を表示。
- **サブカーソル（Δ）トグル ON** で **2本目 (B)** を可視幅 **75%（右寄り）** に追加し、A→B の差分を計算（Δt = tB − tA、信号ごと Δy = y(B) − y(A)）。**B は A（メイン）が ON の間だけ選択可**（A 起点のため。OFF 時はメニュー項目をグレーアウト）。
- **B の表示が差分・範囲統計列の表示/非表示を切り替える**（さらにその統計列内で表示項目を選択式）。
- 既定位置（50%/75%）は出現時のもの。以後は各線の D&D で移動。再表示時は再び既定位置に出す（位置は非永続）。コンテキストメニューに「Reset」項目は設けない（消去＝トグル OFF）。

## 5. 受け入れ要件（親 R14–R17 からの抽出と調整）

親 `valisync-gui/requirements.md`（アーカイブ）の R14–R17 を一次とし、本スコープで以下を**意図的に調整**する（要件逸脱・対話合意済み）:

1. **R15.2「補間値を凡例エリアに表示」**: 既存 `pg.LegendItem` を**撤去**し、**フロート表が凡例を兼ねる**。補間値はフロート表の「値」列に表示。色↔信号名の識別もフロート表が担う（カーソル非表示時は複数 Y 軸の軸ラベルが識別を補完）。
2. **R16.6「最大5組（10本）」→「1パネル1組（2本）」**: 「1プロットに1範囲」方針に整合。多範囲同時比較とその区別 UI は不採用。
3. **R14 起動ジェスチャ**: 要件文の「波形を水平ドラッグ」を、**アクティブ波形（曲線上で押下）＋水平ドラッグ**として具体化（修飾キーではなく曲線ヒットで判別）。
4. **R14.3 適用対象の Session マッピング**: (a) その信号のみ → `apply_offset(signal, signal_offset=Δt)`、(b) 同一 Signal_Group → 同グループ各信号へ `file_offset=Δt`。`Session.group_signals(key)` でグループ信号を取得。
5. **R15.1 起動ジェスチャ（改訂・2026-06-29 合意）**: 「クリックで設置/移動」を**廃止**し、表示 ON/OFF を**右クリックメニューのチェック式トグル**（「メインカーソル」/「サブカーソル（Δ）」）に変更。出現位置は既定（A=50%/B=75%）、移動は線の **D&D のみ**。サブはメイン ON の間だけ選択可。コンテキストメニューの「Reset」項目は**設けない**（消去＝トグル OFF）。※R15 は増分A で「クリック設置」を実装・PR #21 merged のため、本増分でその挙動と realgui クリック設置テストを撤去する。

上記以外（R15.3/5、R16.1–5、R17.1–5、R14.1/2/4/5/6/7/8）は親要件の受け入れ基準をそのまま満たす（R15.1 は項目5の通り改訂）。

## 6. 読み取り面（フロート表）

- **出没**: カーソル表示に連動。カーソル無し→非表示、Global（メイン ON）→表示、Delta（サブ ON）→統計列を追加表示。
- **配置**: プロット上のシーンアイテム（既存凡例と同じ親付け方式）。**ドラッグ移動可**。レイアウトを消費しない。
- **ヘッダ（時刻）・常時表示（2026-06-29 合意）**: 読み取り時刻をヘッダ行に常に出す（**サブ非表示でも A 時刻が見える** — R15.1 改訂で空クリック表示が無くなったぶん、時刻はヘッダで保証）。表記は色ドットで A/B を識別（A=琥珀・B=青＝カーソル線色と一致、「A／B」ラベルは付けない）:
  - Global: `● t_A`（例 `● 0.500 s`）
  - Delta: `● t_A   ● t_B   ·   Δt`（例 `● 0.500 s  ● 0.750 s · Δt 0.250 s`、Δt を白字強調）
- **列構成（8 列フラット・現状維持）**: `●色 ＋ 信号名 ｜ A値`、Delta 時に ＋ `Δy（A→B、符号で色分け）｜ mean ｜ max ｜ min ｜ std ｜ count`。
- **統計項目の選択**: フロート表ヘッダの「列 ▾」小メニューで列の取捨選択（既定＝全列表示）。選択の**永続化はしない**（Phase 3 follow-up）。
- **特殊表示**: 範囲外＝「範囲外」、データなし（count=0）＝「データなし」（§9）。

## 7. アーキテクチャ・コンポーネント

### Core（変更なし・Session 経由のみ）
`apply_offset` / `interpolate` / `compute_statistics` / `unified_timeline_signals` を流用。`interpolate` は範囲外で `None` を返すため R15.5 にそのまま使える。`compute_statistics` は `t_start>t_end` で ValueError を投げるため、VM 側で **A<B に正規化**してから呼ぶ。

### View 層（`GraphPanelView`）
- **カーソルアイテム**: Global (A)・Delta (B) の垂直線（`pg.InfiniteLine`、master ViewBox に追加）。線上ドラッグで移動。A は R15 で実装済み、B を増分B で追加。
- **フロート表（`CursorReadout`）**: R15 で実装済み。増分B で時刻ヘッダ・Δy・統計列・「列 ▾」を `set_readings` 拡張で追加。
- **ジェスチャ**: 空クリック設置は撤去（R15 の `_cursor_press_pos`/`_place_cursor_at` と mousePress/Release の該当分岐を削除）。表示 ON/OFF は `build_context_menu` のチェック式アクションへ。
- **既存凡例 `_legend` は撤去済み（R15）**。

### VM 層
- **`GraphPanelVM`**: 非永続のカーソル状態（A 時刻・Delta 有効・B 時刻・補間方式・統計列選択）。R15 で A 時刻・補間方式・`cursor_readings()` 実装済み。増分B で `delta_enabled`/`cursor_t_b`、Δy・統計（`compute_statistics` 委譲・A<B 正規化）・統計列選択を追加。
- **`GraphAreaVM`**: Global_Cursor 時刻を兄弟パネルへブロードキャスト（R15 `propagate_cursor` 実装済み）。タブ跨ぎは独立。
- **`AppViewModel`**: （R14 用）オフセット状態（`signal_offsets` / `file_offsets` dict）を保持。`apply_offset(signal_key, delta_t, scope)` で dict 更新 ＋ Session 適用 ＋ 通知 → 当該信号を表示する全 `GraphPanelVM` が再描画（R14.5）。

## 8. データフロー

1. **カーソル表示**（右クリック「メインカーソル」トグル ON）: View → `GraphPanelVM` に時刻 A=可視幅50% を設定 → 各信号 `interpolate(signal, A, method)` → フロート表更新（ヘッダに t_A）。
2. **全パネル同期**: `GraphPanelVM` 変更 → `GraphAreaVM` が兄弟パネルへ A をブロードキャスト → 各パネルが自身のレンジ内で A を描画・補間表示（R15.1）。
3. **カーソル移動**（線上 D&D）: リアルタイムに A/B 更新 → 補間/差分/統計を再計算（ドラッグ中はデバウンス/スロットル＝LOD と同じ予算管理）。
4. **Delta**（右クリック「サブカーソル（Δ）」トグル ON）: B=可視幅75% を設定 → Δt/Δy 計算 ＋ `compute_statistics(signal, A, B)`（A<B に正規化）→ 統計列表示（R16/R17）。
5. **オフセット**: 曲線上押下＝アクティブ化 → 水平ドラッグで Δt を計算しアクティブ波形のみプレビュー平行移動（ツールチップに Δt 3 桁、R14.6）→ リリースで適用ダイアログ（(a)信号のみ /(b)グループ/キャンセル）→ `AppViewModel.apply_offset` → 全パネル再描画（R14.4/5）。Escape はプレビュー破棄で開始前へ復元（R14.7/8）。

## 9. エラー処理・エッジケース

- **範囲外（R15.5）**: `interpolate` が `None` → 該当信号を「範囲外」表示、補間しない。
- **データなし（R17.5）**: 範囲内にサンプル無し（`compute_statistics` の count=0・各値 NaN）→ 統計を「データなし」表示。
- **オフセット中断**: Escape（R14.7）/ダイアログのキャンセル（R14.8）→ プレビューを破棄し offset dict を変更しない。
- **A==B**: Δt=0・統計範囲ゼロ → 「データなし」相当。
- **空パネル/信号なし**: カーソル・オフセットは無反応（操作対象なし）。
- **アクティブ波形がドラッグ中に除去**: アクティブ解除しプレビュー破棄。

## 10. テスト戦略（GUI テストレイヤー）

`docs/gui-testing-layers.md` 準拠。計画時 `/gui-test-plan`、merge 前 `/gui-verify`。

- **Layer A（VM/ロジック）**: 補間値・Δt/Δy・統計（A<B 正規化・範囲外/データなし）・offset dict 更新とスコープ分岐（signal_offset / file_offset）・ブロードキャスト先選定・トグル状態遷移（メイン OFF→B 不可）。

- **Layer C（realgui・必須）**: 実 OS 入力で ①メイン/サブ トグル→各線の出現（既定位置）と補間値・時刻ヘッダ、②線上 D&D→カーソル移動と統計/Δ 再計算、③曲線ドラッグ→オフセットプレビュー＋適用ダイアログ＋全パネル更新（R14）。スクリーンショットは `QT_QPA_PLATFORM=windows`（offscreen は文字が□化）。InfiniteLine の D&D は scene ドラッグ（QDrag 無し）でメインスレッド駆動可、R14 の曲線ドラッグ（QDrag 系）は別 OS スレッド＋watchdog。

## 11. 既存仕様・成果物との関係

- **PR #19 操作モデルと一貫**: アクティブ軸（Y ストリップ）とアクティブ波形（プロット内）は同じ「選択してから操作」哲学。Y/X ストリップの既存責務は不変。
- **凡例撤去は MVP（R13 相当）への変更**（R15 で実施済み）: 識別情報がフロート表＋軸ラベルへ移ることを明記（情報欠落なし）。

- **`.kiro/specs/valisync-gui-analysis` は未作成**（アーカイブ側に分割前の親要件のみ）。本設計が一次情報源。親 `valisync-gui/requirements.md` はアーカイブで編集しない。
- **roadmap / CLAUDE.md Phase 表**: 実装完了時に状態を更新（トレーサビリティ）。

## 12. 実装時に確定する細部

**増分B で確定（2026-06-29 合意）:**
- カーソル表示 ON/OFF＝右クリックのチェック式トグル（メイン/サブ）、移動＝線の D&D のみ、空クリック設置は廃止。
- 既定出現位置: メイン A=可視幅50%・サブ B=75%（非永続、再表示で再び既定位置）。サブはメイン ON の間だけ選択可。
- 読み取り面ヘッダに時刻を常時表示（色ドット識別＝カーソル線色と一致、Δt 強調）。統計表は 8 列フラット維持、「列 ▾」で取捨選択。コンテキストメニューに「Reset」は設けない。
- Global_Cursor ブロードキャストは **常時**（"Sync X" トグル非依存。各パネルは自レンジ内で時刻 A を描画）。
