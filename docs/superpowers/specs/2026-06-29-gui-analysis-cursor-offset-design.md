## 1. 背景・目的

### 現状（実コードで確認）
- **R14（時間オフセット）/ R15（Global_Cursor）/ R16（Delta_Cursor）/ R17（範囲統計）は GUI に未実装**。`src/valisync/gui` に `InfiniteLine`・カーソル・`apply_offset` 呼び出し・`compute_statistics` 呼び出し・統一タイムライン配線は存在しない（grep 確認: offset 系のヒットは軸リサイズの `grip_offset` のみ）。
- **コア API は充足**: `Session.apply_offset(signal, file_offset=0.0, signal_offset=0.0) -> Signal`、`Session.interpolate(signal, t, method=LINEAR) -> float | None`、`Session.compute_statistics(signal, t_start, t_end) -> StatisticsResult`、`Session.unified_timeline_signals(file_offsets, signal_offsets)`。
- **既存凡例**: `GraphPanelView` は `pg.LegendItem`（`graph_panel_view.py:848`、master ViewBox に親付け）で信号名を表示している。
- **プロット内側ジェスチャは空き**: `GraphPanelView.mousePressEvent`（`:1180`）は X 軸ストリップのゾーン（`ZONE_X_INNER/OUTER`）でしかドラッグを開始しない。Y 操作は `_AlignedAxisItem`（アクティブ軸モデル・PR #19）が所有し、ViewBox は `setMouseEnabled(False)`。**プロット描画領域（`ZONE_PLOT`）の素のクリック/ドラッグは未使用**。
- **全パネル同期の足場**: `GraphAreaVM` がタブごとに `panels: list[GraphPanelVM]` と `x_sync_enabled` を保持し、兄弟パネルへ X レンジをブロードキャストする機構（"Sync X"）を持つ。

### 目的
親 R14–R17 の受け入れ基準を満たす解析操作（カーソル計測・範囲統計・時間オフセット）を、**PR #19 の「選択してから操作」操作モデルと一貫**させ、**空いているプロット内側ジェスチャ空間**に衝突なく載せる。コアは Session 経由のみで呼ぶ（MVVM）。

## 3. 用語

| 用語 | 意味 |
|---|---|
| Global_Cursor | パネルに1本の垂直線。時刻 A を指し、各信号の補間値を表示。クリックで設置/移動。 |
| Delta_Cursor | Delta 表示時に追加される2本目の垂直線（時刻 B）。A→B の差分を計測。 |
| アクティブ波形 | オフセット操作の対象として掴まれた1曲線（`_items` の1キー＝信号名）。PR #19 のアクティブ軸に相当する非永続 UI 状態。 |
| フロート表 | プロット上にオーバーレイする読み取り面。カーソル表示に連動して出没。凡例（色↔信号名）を兼ねる。 |
| 統一タイムライン | `file_offsets`/`signal_offsets` を適用して時刻同期した共通時間軸。 |

## 4. 操作モデル（プロット内側ジェスチャの交通整理）

既存責務（左 Y ストリップ＝アクティブ軸操作・下 X ストリップ＝常時ズーム/パン）は**変更しない**。プロット描画領域の押下を「押した先」で振り分ける（新規ディスパッチャ）:

| 押下位置 → 動作 | 内容 | 要件 |
|---|---|---|
| **空白でクリック（移動なし）** | Global_Cursor を設置/移動 | R15.1 |
| **カーソル線上でドラッグ** | その線（A または B）を移動 | R15.4 / R16.4 |
| **曲線上で押下＋水平ドラッグ** | その曲線をアクティブ化し時間オフセット（プレビュー）。リリースで適用ダイアログ | R14.1 |
| **右クリック** | コンテキストメニュー：Delta 表示切替・補間方式・Reset（統計列の選択はフロート表上メニュー＝§6） | R15.4 |

- **判定優先順位**: カーソル線（許容ピクセル内）＞ 曲線（最近傍・許容ピクセル内）＞ 空白。
- **クリック vs ドラッグ**: 移動しきい値未満＝クリック（カーソル設置）、しきい値超＝ドラッグ（線移動 or オフセット）。最頻のカーソル設置を修飾なしに保ち、データ改変系（オフセット）は曲線上ドラッグに限定して誤爆を防ぐ。
- 曲線ヒットテストの正確な許容値・アクティブ波形のハイライト表現は**実装時に確定**（§12）。

### カーソルモデル（加算式）
- **Global_Cursor (A)** が起点。カーソル表示中は常に A における各信号の補間値を表示。
- **Delta 表示**を有効化すると **2本目 (B)** を追加し、A→B の差分を計算（Δt = tB − tA、信号ごと Δy = y(B) − y(A)）。
- **B の表示が差分・範囲統計列の表示/非表示を切り替える**（さらにその統計列内で表示項目を選択式）。

## 5. 受け入れ要件（親 R14–R17 からの抽出と調整）

親 `valisync-gui/requirements.md`（アーカイブ）の R14–R17 を一次とし、本スコープで以下を**意図的に調整**する（要件逸脱・対話合意済み）:

1. **R15.2「補間値を凡例エリアに表示」**: 既存 `pg.LegendItem` を**撤去**し、**フロート表が凡例を兼ねる**。補間値はフロート表の「値」列に表示。色↔信号名の識別もフロート表が担う（カーソル非表示時は複数 Y 軸の軸ラベルが識別を補完）。
2. **R16.6「最大5組（10本）」→「1パネル1組（2本）」**: 「1プロットに1範囲」方針に整合。多範囲同時比較とその区別 UI は不採用。
3. **R14 起動ジェスチャ**: 要件文の「波形を水平ドラッグ」を、**アクティブ波形（曲線上で押下）＋水平ドラッグ**として具体化（修飾キーではなく曲線ヒットで判別）。
4. **R14.3 適用対象の Session マッピング**: (a) その信号のみ → `apply_offset(signal, signal_offset=Δt)`、(b) 同一 Signal_Group → 同グループ各信号へ `file_offset=Δt`。`Session.group_signals(key)` でグループ信号を取得。

上記以外（R15.1/3/5、R16.1–5、R17.1–5、R14.1/2/4/5/6/7/8）は親要件の受け入れ基準をそのまま満たす。

## 6. 読み取り面（フロート表）

- **出没**: カーソル表示に連動。カーソル無し→非表示、Global 表示→表示、Delta 表示→統計列を追加表示。
- **配置**: プロット上のシーンアイテム（既存凡例と同じ親付け方式）。**ドラッグ移動可**。レイアウトを消費しない。
- **列構成**:
  - 常時（Global）: ●色 ＋ 信号名 ＋ 値（A の補間値）
  - Delta 時: ＋ Δy（A→B）＋ 範囲統計列（mean/max/min/std/count）
- **統計項目の選択**: フロート表上の小メニューで列の取捨選択（既定＝全列表示）。選択の**永続化はしない**（Phase 3 follow-up）。
- **特殊表示**: 範囲外＝「範囲外」、データなし＝「データなし」（§9）。

## 7. アーキテクチャ・コンポーネント

### Core（変更なし・Session 経由のみ）
`apply_offset` / `interpolate` / `compute_statistics` / `unified_timeline_signals` を流用。`interpolate` は範囲外で `None` を返すため R15.5 にそのまま使える。

### View 層（`GraphPanelView`）
- **カーソルアイテム**: Global (A)・Delta (B) の垂直線（`pg.InfiniteLine` 相当、master ViewBox に追加）。線上ドラッグで移動。
- **フロート表**: 新規オーバーレイウィジェット（シーンアイテム）。`render_data()` の色・信号名と、VM から得た補間値/差分/統計を表示。
- **ジェスチャディスパッチャ**: `mousePressEvent` を拡張し `ZONE_PLOT` 押下を §4 表で振り分け（既存 X/Y ゾーン処理は不変）。`_data_value(pos, "x")`（`:1167`）でピクセル→時刻変換を流用。
- **既存凡例 `_legend` を撤去**（`addItem` 呼び出し・生成・`legend_labels` を除去/置換）。

### VM 層
- **`GraphPanelVM`**: 非永続のカーソル状態（A 時刻・Delta 有効・B 時刻・補間方式・統計列選択）。補間値/Δ/統計を Session 経由で計算して View へ供給するメソッド群を追加。
- **`GraphAreaVM`**: Global_Cursor 時刻を**アクティブタブ内の兄弟パネルへブロードキャスト**（既存 X-sync 機構に相乗り）。タブ跨ぎは独立。
- **`AppViewModel`**: **オフセット状態（`signal_offsets` / `file_offsets` dict）を保持**。`apply_offset(signal_key, delta_t, scope)` で dict 更新 ＋ Session 適用 ＋ 通知 → 当該信号を表示する全 `GraphPanelVM` が再描画（R14.5）。オフセット適用後の時刻は統一タイムライン経由で全ビューに反映。

## 8. データフロー

1. **カーソル設置**（空白クリック）: View → `GraphPanelVM` に時刻 A 設定 → 各信号 `interpolate(signal, A, method)` → フロート表更新。
2. **全パネル同期**: `GraphPanelVM` 変更 → `GraphAreaVM` が兄弟パネルへ A をブロードキャスト → 各パネルが自身のレンジ内で A を描画・補間表示（R15.1）。
3. **カーソル移動**（線上ドラッグ）: リアルタイムに A/B 更新 → 補間/差分/統計を再計算（ドラッグ中はデバウンス/スロットル＝LOD と同じ予算管理）。
4. **Delta**: B 設定 → Δt/Δy 計算 ＋ `compute_statistics(signal, A, B)`（A<B に正規化）→ 統計列表示（R16/R17）。
5. **オフセット**: 曲線上押下＝アクティブ化 → 水平ドラッグで Δt を計算しアクティブ波形のみプレビュー平行移動（ツールチップに Δt 3 桁、R14.6）→ リリースで適用ダイアログ（(a)信号のみ /(b)グループ/キャンセル）→ `AppViewModel.apply_offset` → 全パネル再描画（R14.4/5）。Escape はプレビュー破棄で開始前へ復元（R14.7/8）。

## 9. エラー処理・エッジケース

- **範囲外（R15.5）**: `interpolate` が `None` → 該当信号を「範囲外」表示、補間しない。
- **データなし（R17.5）**: 範囲内にサンプル無し → 統計を「データなし」表示。
- **オフセット中断**: Escape（R14.7）/ダイアログのキャンセル（R14.8）→ プレビューを破棄し offset dict を変更しない。
- **A==B**: Δt=0・統計範囲ゼロ → 「データなし」相当。
- **空パネル/信号なし**: カーソル・オフセットは無反応（操作対象なし）。
- **アクティブ波形がドラッグ中に除去**: アクティブ解除しプレビュー破棄。

## 10. テスト戦略（GUI テストレイヤー）

`docs/gui-testing-layers.md` 準拠。計画時 `/gui-test-plan`、merge 前 `/gui-verify`。

- **Layer A（VM/ロジック）**: 補間値・Δt/Δy・統計（A<B 正規化・範囲外/データなし）・offset dict 更新とスコープ分岐（signal_offset / file_offset）・ブロードキャスト先選定・ジェスチャ振り分け関数（押下位置→動作）。
- **Layer B（ヘッドレス結合）**: カーソルアイテムの生成/可視、フロート表の出没と列構成、凡例撤去の確認。※ **実イベント経路・カーソル形状・ドラッグプレビュー・クロスパネル実配送は Layer B では証明不可**。
- **Layer C（realgui・必須）**: 実 OS 入力で ①空白クリック→カーソル設置と補間値、②線上ドラッグ→カーソル移動と全パネル同期、③曲線ドラッグ→オフセットプレビュー＋適用ダイアログ＋全パネル更新、④Delta 表示→統計列の出没。スクリーンショットは `QT_QPA_PLATFORM=windows`（offscreen は文字が□化）。ドラッグ駆動は別 OS スレッド＋watchdog（QDrag/モーダルループのハング回避・memory 参照）。

## 11. 既存仕様・成果物との関係

- **PR #19 操作モデルと一貫**: アクティブ軸（Y ストリップ）とアクティブ波形（プロット内）は同じ「選択してから操作」哲学。Y/X ストリップの既存責務は不変。
- **凡例撤去は MVP（R13 相当）への変更**: 既存機能の削除を含むため、識別情報がフロート表＋軸ラベルへ移ることを明記（情報欠落なし）。
