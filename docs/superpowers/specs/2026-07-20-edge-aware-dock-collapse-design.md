# 辺対応の折りたたみ（edge-aware-dock-collapse）設計

日付: 2026-07-20 ／ ステータス: 承認済み（brainstorming）
出自: **増分C（折りたたみ可能ドック・PR #131 merged）の手直し**。実機確認でユーザーが
「右ドックが横のまま薄くなるのは想定と違う。畳む方向をドックの接している辺に合わせたい」
と判明。目標イメージはモック3回反復で確定（辺沿いの gutter＋content サイズのコンパクトタブ）。

## 1. 背景・問題

増分C（PR #131）の `CollapsibleDockTitleBar` は畳む操作が**常に高さクランプ（横帯）**で、
右ドック（File/Channel）も横幅を保ったまま薄い帯になり、横スペースが空かない。ユーザーの
意図は「畳む方向をドックが接している辺に合わせる」= VS Code / JetBrains のツールウィンドウ
畳みと同型:

- **右/左ドック** → 幅を詰めて、その辺に**全高の細い縦レール**が出る。中に**縦書きタイトル**の
  content サイズの縦タブ。空いた横スペースは中央プロットが取り戻す
- **下ドック** → 高さを詰めて、**全幅の横帯**の中に content サイズの横チップ（左寄せ）
- **帯（レール/gutter）は縁いっぱい（全高/全幅）を確保するが、中のタブは全幅/全高に
  引き伸ばさない**（content サイズ）

現行の「タイトルバーを高さクランプ」方式は QDockWidget のタイトルバーが常に上端の水平
ストリップである制約と噛み合わず、全高の縦レールにならない。よって畳みの**機構自体**を
「クランプ」から「dock を hide して辺ごとのレールにコンパクトタブを出す」方式へ差し替える。

## 2. 要件（確定した設計判断）

- **畳む方向・場所は「ドックの種類」ではなく「現在配置されている辺（位置）」で動的決定**。
  畳む時点の `dockWidgetArea(dock)` を見て、左/右＝縦レール・下＝横帯を選ぶ。ドックを別辺へ
  動かせば畳み方も追従する
- **上端への配置を構造的に禁止**。3ドックに `setAllowedAreas(Left | Right | Bottom)` を設定
  し Top を許可領域から外す（上端へドラッグしても弾かれる）。→ 上用レールは作らず、上端の
  边界ケースも消滅。対応辺 = **{左・右・下}**
- **畳んだタブはクリックで展開のみ**（フロート/閉じるは載せない・展開後のタイトルバーで操作）
- **レールは辺ごとに1本・遅延生成**（その辺に畳んだドックが出たとき初めて現れ、空になったら
  隠してスペースを占有しない）
- **タブは content サイズ**（引き伸ばさない）。左/右＝**上寄せ**・下＝**左寄せ**。複数は
  ドックの辺上の位置順で積む（クリック順で入れ替わらない）
- **ツールチップ（ホバー説明）なし**
- **新トークンなし**（配色は既存 chrome トークン流用）。追加は `chevron_left`・`chevron_up`
  アイコンの vendoring のみ（アイコン追加はトークン不変方針の範囲内）
- **既存の QSettings 永続（`dockCollapsed`）・realgui Layer C・トークン不変方針を継承**
- **スコープ外**: 診断絵文字グリフの SVG 置換・比較データモデル等

## 3. アーキテクチャ（§1）

**新コンポーネント `DockCollapseRail`（辺ごとの畳みレール・新規1ファイル `gui/views/`）**:
辺沿いの細いストリップ widget。左/右＝縦（狭い幅・辺のドック領域の全高）、下＝横（低い
高さ・全幅）。中央側に境界線を1本引き gutter として視認させる（chrome トークン流用）。
中に content サイズのタブボタンを積む。空のときは自身を隠す。

**タブ**:
- 左/右（縦タブ）: 縦書きタイトル＋展開シェブロン。上寄せで、ドックの元の上下順で積む
- 下（横チップ）: シェブロン＋タイトルを左寄せ
- シェブロンは開く方向を指す: 右レール→ `chevron_left`／左レール→ `chevron_right`／
  下レール→ `chevron_up`
- クリックで対応ドックを展開

**畳む**（展開時タイトルバーの chevron クリック）:
1. `dock.hide()`（→ 中央が空いたスペースを取り戻す）
2. 畳む時点の `dockWidgetArea(dock)` で辺判定 → その辺のレール（無ければ生成）に
   ドックの位置順でタブを挿入。畳む直前の辺方向 extent（右/左＝幅・下＝高さ）を控える

**展開**（タブ本体クリック）:
1. レールからそのタブを除去し `dock.show()`
2. 控えた extent へ `resizeDocks` で復元（軸はドックの辺に対応）。レールが空になったら隠す

**既存 `CollapsibleDockTitleBar` は「展開時のタイトルバー」として継続利用**。chevron の動作
だけ「高さクランプ」→「hide＋レールにタブ追加」へ差し替える（フロート/閉じるは不変）。

**MainWindow 側**: 3ドックに `setAllowedAreas`。辺ごとのレールを保持し、collapse/expand の
配線・辺判定・タブ↔ドック対応・QSettings 永続をまとめる。

**レール widget の実現方式はスパイクで確定**（§5）。QDockWidget レイアウトの finicky さ
（FU-14 defer の所以）と `setCorner`（FU-10）干渉があるため、机上で決めず実機で選ぶ。

## 4. 既存機構との整合（§3）

- **QSettings 永続**: 既存 `dockCollapsed`（`{objectName: bool}`）を継続。復元時は
  `_apply_saved_collapse` が「畳み状態のドックを hide＋対応辺レールにタブ追加」へ差し替え。
  復元順は現状どおり `_restore_state()`（`restoreState`）の後に再適用（corner 再適用と同型・
  memory gui_restorestate_resets_dock_corner_config）
- **フロート/閉じる**: 展開時タイトルバーの機能として維持。畳んだタブには載せない。畳んだ
  状態で閉じたいときは一度展開してから
- **Reset Layout**（`_reset_layout`）: 現状の「全展開へ」を継承。全ドックをレールから戻して
  展開＋レールを空にする
- **復元サイズ**: 畳む直前の辺方向 extent を控え、展開時にその軸へ `resizeDocks`（現行の
  高さ復元を軸対応へ一般化）。控えが無い初回は既定値（幅/高さそれぞれ）
- **フロート中のドックを畳む**: フロート中は送り先の辺が無いのでレールに送れない。畳み
  トグル（chevron）はドッキング時のみ有効・**フロート中は無効化（グレーアウト）**（非表示に
  すると再ドッキング後に消えたままに見える事故を避ける）。ドックが再ドッキングされたら有効化

## 5. スパイク・検証（§4）

**スパイク（プラン Task 1・実機必須）**: レール widget の実現方式を着手前に確定。候補:
- (a) 右/下/左の QToolBar 領域（Qt ネイティブの縦/横ストリップ）
- (b) 中央 widget を包む自作エッジストリップ
- (c) 常設の細い QDockWidget をレール化

実機で「hide＋レールで**中央に実際にスペースが空くか**・レールが正しい全長で出るか・
`setCorner`(FU-10) と干渉しないか・縦書きタブが描けるか」を比較し1案採用＋フォールバック記録。

**Layer B（CI）**:
- レールの遅延生成/破棄・畳む＝dock hide＋タブ追加・展開＝タブ除去＋dock show＋サイズ復元
- 辺判定が正しいレールへ写像・`setAllowedAreas` が Top を拒否
- `dockCollapsed` ラウンドトリップ（畳み→save→新インスタンスで復元→畳まれたまま）
- `_reset_layout` で全展開・積み順の安定（位置順・クリック順で入れ替わらない）

**Layer C（realgui・/gui-verify ①ゲート）**: **畳みの実効は実機でしか確証できない**
（memory gui_isvisible_true_for_offscreen_hidden_dock・gui_dock_toggle_width_change_needs_real_display_and_layout）。
実 OS クリックで右ドックを畳む→**ドック非表示＋右レールに縦タブ出現＋中央プロットが実際に
広がる**（中央幅の増加を実測）／タブクリックで元幅へ復元。下ドックは横帯で同様。フロート/
閉じるが展開時に実動作＋journey smoke。

## 6. 成果物（§4）

- 畳んだ姿が変わる（旧・全幅帯→縦レール＋縦タブ / 横帯＋左寄せチップ）＝凍結カタログの
  `09_collapsed` を撮り直し。前後差分で「畳み表現の変化のみ・展開時の通常状態は不変」を確認
  → ベースライン差し替え・カタログ再撮影・DesignSync 再同期（Ground Truth 更新）
- **トークン/エクスポート変更なし**（tokens.css/json/cards 不変）。追加は `chevron_left`/
  `chevron_up` アイコン（Icons カード overview は既存＋2アイコンに更新）
- docs/design.md 決定履歴に運用反復（増分C 手直し・辺対応の折りたたみ・トークン変更なし・
  出典＝実機確認でのユーザー指摘）。CLAUDE.md 更新は merge 後 docs PR
